## What Are Service Tests?

### Definition

Service tests verify your **business logic** layer:

-   CRUD operations work correctly
-   Validation is enforced
-   Correct exceptions are raised
-   Database transactions work

### Position in Testing Pyramid

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    Service Tests in the Pyramid                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│                            /\                                            │
│                           /  \                                           │
│                          / E2E\                                          │
│                         /──────\                                         │
│                        /        \                                        │
│                       / Service  \  ← We are here                       │
│                      /   Tests    \   (Integration)                      │
│                     /──────────────\                                     │

│                   /   Model Tests    \                                   │
│                  /______________________\                                │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

## Service Tests vs Model Tests

| Aspect           | Model Tests      | Service Tests          |
| ---------------- | ---------------- | ---------------------- |
| **Focus**        | Model behavior   | Business logic         |
| **Scope**        | Single model     | Service + models + DB  |
| **Dependencies** | Minimal          | Real database          |
| **Example**      | Password hashing | User registration flow |
| **Location**     | `test_models/`   | `test_services/`       |

### Example Comparison

```python
# Model test - tests User model directly
def test_password_hashing(self, db):
    user = User(username="test", email="test@example.com")
    user.set_password("password123")
    assert user.check_password("password123") is True

# Service test - tests UserService.create_user()
def test_create_user_success(self, db):
    data = UserCreateSchema(username="test", email="test@example.com", password="password123")
    user = UserService.create_user(data)
    assert user.id is not None
    assert user.check_password("password123") is True
```

## Test Categories

### Category 1: Create Operations

```python
class TestUserServiceCreate:
    """Tests for user creation service."""

    def test_create_user_success(self, db):
        """Test successful user creation."""
        # ARRANGE - Prepare input data
        data = UserCreateSchema(
            username="serviceuser",
            email="service@example.com",
            password="SecurePass123!",
            first_name="Service",
            last_name="User",
        )

        # ACT - Call the service
        user = UserService.create_user(data)

        # ASSERT - Verify the result
        assert user.id is not None
        assert user.username == "serviceuser"
        assert user.email == "service@example.com"
        assert user.check_password("SecurePass123!")

    def test_create_user_duplicate_username(self, db, sample_user):
        """Test creating user with duplicate username raises error."""
        data = UserCreateSchema(
            username="testuser",  # Same as sample_user
            email="different@example.com",
            password="SecurePass123!",
            first_name="Test",
        )

        with pytest.raises(UserAlreadyExistsError) as exc_info:
            UserService.create_user(data)

        assert exc_info.value.field == "username"
```

**What these test:**

-   Happy path: User is created successfully
-   Error path: Duplicate data raises appropriate exception
-   Exception attributes: Field name is set correctly

### Category 2: Authentication

```python
class TestUserServiceAuthentication:
    """Tests for user authentication service."""

    def test_authenticate_success(self, db, sample_user):
        """Test successful authentication."""
        user = UserService.authenticate("test@example.com", "TestPassword123!")
        assert user.id == sample_user.id

    def test_authenticate_wrong_password(self, db, sample_user):
        """Test authentication with wrong password."""
        with pytest.raises(InvalidCredentialsError):
            UserService.authenticate("test@example.com", "WrongPassword!")

    def test_authenticate_user_not_found(self, db):
        """Test authentication with non-existent user."""
        with pytest.raises(UserNotFoundError):
            UserService.authenticate("nobody@example.com", "AnyPassword123!")
```

**What these test:**

-   Correct credentials return user
-   Wrong password raises `InvalidCredentialsError`
-   Non-existent user raises `UserNotFoundError`

### Category 3: Read Operations

```python
class TestUserServiceRead:
    """Tests for user retrieval services."""

    def test_get_by_id(self, db, sample_user):
        """Test getting user by ID."""
        user = UserService.get_by_id(sample_user.id)
        assert user.id == sample_user.id
        assert user.username == "testuser"

    def test_get_by_id_not_found(self, db):
        """Test getting non-existent user returns None."""
        user = UserService.get_by_id(99999)
        assert user is None

    def test_get_by_id_or_404(self, db, sample_user):
        """Test get_by_id_or_404 returns user."""
        user = UserService.get_by_id_or_404(sample_user.id)
        assert user.id == sample_user.id

    def test_get_by_id_or_404_raises(self, db):
        """Test get_by_id_or_404 raises for non-existent user."""
        with pytest.raises(UserNotFoundError):
            UserService.get_by_id_or_404(99999)

    def test_get_all_with_pagination(self, db, sample_users):
        """Test getting all users with pagination."""
        users, total = UserService.get_all(page=1, per_page=2)
        assert len(users) == 2
        assert total == 5

    def test_get_all_excludes_deleted(self, db, sample_user):
        """Test that soft-deleted users are excluded by default."""
        sample_user.soft_delete()
        db.session.commit()

        users, total = UserService.get_all()
        assert total == 0

    def test_get_all_includes_deleted(self, db, sample_user):
        """Test including soft-deleted users."""
        sample_user.soft_delete()
        db.session.commit()

        users, total = UserService.get_all(include_deleted=True)
        assert total == 1
```

**What these test:**

-   Various retrieval methods work
-   Pagination returns correct counts
-   Soft-delete filtering works

### Category 5: Delete Operations

```python
class TestUserServiceDelete:
    """Tests for user deletion service."""

    def test_soft_delete(self, db, sample_user):
        """Test soft delete."""
        user = UserService.soft_delete(sample_user.id)
        assert user.is_deleted is True

    def test_soft_delete_not_found(self, db):
        """Test soft deleting non-existent user."""
        with pytest.raises(UserNotFoundError):
            UserService.soft_delete(99999)

    def test_restore(self, db, sample_user):
        """Test restoring soft-deleted user."""
        sample_user.soft_delete()
        db.session.commit()

        user = UserService.restore(sample_user.id)
        assert user.is_deleted is False

    def test_hard_delete(self, db, sample_user):
        """Test hard delete."""
        user_id = sample_user.id
        result = UserService.hard_delete(user_id)

        assert result is True
        assert UserService.get_by_id(user_id, include_deleted=True) is None
```

**What these test:**

-   Soft delete marks as deleted
-   Restore reverses soft delete
-   Hard delete removes from database

## Testing Exceptions

### Pattern: Test Exception Type

```python
def test_authenticate_wrong_password(self, db, sample_user):
    """Test that wrong password raises InvalidCredentialsError."""
    with pytest.raises(InvalidCredentialsError):
        UserService.authenticate("test@example.com", "WrongPassword!")
```

### Pattern: Test Exception Attributes

```python
def test_create_user_duplicate_email(self, db, sample_user):
    """Test exception contains correct field."""
    with pytest.raises(UserAlreadyExistsError) as exc_info:
        UserService.create_user(duplicate_data)

    # Check exception attributes
    assert exc_info.value.field == "email"
    assert exc_info.value.value == "test@example.com"
```

### Pattern: Test Exception Message

```python
def test_user_not_found_message(self, db):
    """Test exception message is descriptive."""
    with pytest.raises(UserNotFoundError) as exc_info:
        UserService.get_by_id_or_404(99999)

    assert "99999" in str(exc_info.value)
```

## Best Practices

### 1. Use Pydantic Schemas

```python
# Good - Use the same schemas as your routes
from app.schemas.user import UserCreateSchema, UserUpdateSchema

def test_create_user(self, db):
    data = UserCreateSchema(
        username="test",
        email="test@example.com",
        password="SecurePass123!"
    )
    user = UserService.create_user(data)
```

### 2. Test Both Success and Failure

```python
# Always test happy path
def test_get_user_success(self, db, sample_user):
    user = UserService.get_by_id(sample_user.id)
    assert user is not None

# And error paths
def test_get_user_not_found(self, db):
    user = UserService.get_by_id(99999)
    assert user is None
```

### 3. Verify Database State

```python
def test_update_persists(self, db, sample_user):
    """Verify changes are actually saved to database."""
    data = UserUpdateSchema(first_name="Updated")
    UserService.update_user(sample_user.id, data)

    # Query fresh from database
    fresh_user = db.session.get(User, sample_user.id)
    assert fresh_user.first_name == "Updated"
```

### 4. Test Edge Cases

```python
def test_empty_update(self, db, sample_user):
    """Update with no changes should work."""
    data = UserUpdateSchema()  # No fields to update
    user = UserService.update_user(sample_user.id, data)
    assert user.id == sample_user.id  # Should still work

def test_pagination_empty(self, db):
    """Pagination with no results."""
    users, total = UserService.get_all()
    assert users == []
    assert total == 0
```

### 5. Group Related Tests

```python
class TestUserServiceCreate:
    """All creation tests together."""
    def test_create_user_success(self, db): ...
    def test_create_user_duplicate_username(self, db): ...
    def test_create_user_duplicate_email(self, db): ...

class TestUserServiceAuthentication:
    """All auth tests together."""
    def test_authenticate_success(self, db): ...
    def test_authenticate_wrong_password(self, db): ...
```

## Summary

### Test Coverage for UserService

| Method             | Tests                               |
| ------------------ | ----------------------------------- |
| `create_user`      | Success, duplicate username/email   |
| `authenticate`     | Success, wrong password, not found  |
| `get_by_id`        | Found, not found                    |
| `get_by_id_or_404` | Found, raises error                 |
| `get_all`          | Pagination, exclude/include deleted |
| `update_user`      | Success, not found                  |
| `soft_delete`      | Success, not found                  |
| `restore`          | Success                             |
| `hard_delete`      | Success                             |

### Key Takeaways

1. **Test with real database** - Services need database integration
2. **Use Pydantic schemas** - Same as your routes
3. **Test exceptions thoroughly** - Type, attributes, message
4. **Group by operation** - Create, Read, Update, Delete
5. **Cover edge cases** - Empty results, not found, etc.
