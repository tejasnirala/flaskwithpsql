## üî§ Basic Syntax Differences

### Semicolons and Braces

**JavaScript:**

```javascript
if (x > 5) {
    console.log("Big");
} else {
    console.log("Small");
}
```

**Python:**

```python
if x > 5:
    print("Big")
else:
    print("Small")
```

**Key differences:**

-   ‚ùå No semicolons (optional, discouraged)
-   ‚ùå No curly braces `{}`
-   ‚úÖ Colons `:` after conditions/loops/functions
-   ‚úÖ **Indentation is required** (usually 4 spaces)

---

### Variables

**JavaScript:**

```javascript
let name = "John"; // Mutable
const age = 25; // Immutable
var oldStyle = "avoid"; // Old style
```

**Python:**

```python
name = "John"            # All variables are mutable
age = 25                 # Convention: UPPERCASE for constants
AGE = 25                 # This is a "constant" (but Python won't enforce it)
```

**Key differences:**

-   No `let`, `const`, `var` keywords
-   No built-in constant enforcement (use UPPERCASE by convention)
-   Variable type is inferred automatically

---

### Printing/Logging

**JavaScript:**

```javascript
console.log("Hello");
console.log("Name:", name, "Age:", age);
console.log(`Template: ${name} is ${age}`);
```

**Python:**

```python
print("Hello")
print("Name:", name, "Age:", age)       # Multiple args, auto-spaced
print(f"Template: {name} is {age}")     # f-string (Python 3.6+)
```

## üì¶ Arrays vs Lists

### Creating

**JavaScript:**

```javascript
const arr = [1, 2, 3];
const arr2 = new Array(5).fill(0);
```

**Python:**

```python
arr = [1, 2, 3]
arr2 = [0] * 5                    # [0, 0, 0, 0, 0]
arr3 = [0 for _ in range(5)]      # Same thing, list comprehension
```

### Common Operations

| Operation       | JavaScript             | Python                    |
| --------------- | ---------------------- | ------------------------- |
| Length          | `arr.length`           | `len(arr)`                |
| Add to end      | `arr.push(4)`          | `arr.append(4)`           |
| Remove from end | `arr.pop()`            | `arr.pop()`               |
| Add to start    | `arr.unshift(0)`       | `arr.insert(0, value)`    |
| Find index      | `arr.indexOf(2)`       | `arr.index(2)`            |
| Check exists    | `arr.includes(2)`      | `2 in arr`                |
| Slice           | `arr.slice(1, 3)`      | `arr[1:3]`                |
| Map             | `arr.map(x => x*2)`    | `[x*2 for x in arr]`      |
| Filter          | `arr.filter(x => x>2)` | `[x for x in arr if x>2]` |
| Join            | `arr.join(", ")`       | `", ".join(arr)`          |

### Slicing (Super Powerful in Python!)

```python
arr = [0, 1, 2, 3, 4, 5]

arr[1:4]      # [1, 2, 3] - from index 1 to 3 (4 excluded)
arr[:3]       # [0, 1, 2] - first 3 elements
arr[3:]       # [3, 4, 5] - from index 3 to end
arr[-1]       # 5 - last element
arr[-2:]      # [4, 5] - last 2 elements
arr[::2]      # [0, 2, 4] - every 2nd element
arr[::-1]     # [5, 4, 3, 2, 1, 0] - reversed!
```

## üóÇÔ∏è Objects vs Dictionaries

### Creating

**JavaScript:**

```javascript
const user = {
    name: "John",
    age: 25,
    "with-dash": "value",
};
```

**Python:**

```python
user = {
    "name": "John",      # Keys MUST be quoted (strings)
    "age": 25,
    "with-dash": "value"
}
```

**‚ö†Ô∏è Important:** In Python dictionaries, keys must be strings (quoted). No shorthand like JS!

### Accessing Values

**JavaScript:**

```javascript
user.name; // "John"
user["name"]; // "John"
user.nonexistent; // undefined
```

**Python:**

```python
user["name"]            # "John"
user.get("name")        # "John" (safer, returns None if missing)
user["nonexistent"]     # KeyError! üí•
user.get("nonexistent") # None (safe)
user.get("nonexistent", "default")  # "default"
```

### Common Operations

| Operation   | JavaScript            | Python                             |
| ----------- | --------------------- | ---------------------------------- |
| Get keys    | `Object.keys(obj)`    | `obj.keys()` or `list(obj.keys())` |
| Get values  | `Object.values(obj)`  | `obj.values()`                     |
| Get entries | `Object.entries(obj)` | `obj.items()`                      |
| Check key   | `"key" in obj`        | `"key" in obj`                     |
| Delete key  | `delete obj.key`      | `del obj["key"]`                   |
| Merge       | `{...obj1, ...obj2}`  | `{**obj1, **obj2}`                 |

## üîÅ Loops

### For Loop

**JavaScript:**

```javascript
// Classic
for (let i = 0; i < 10; i++) {
    console.log(i);
}

// For...of (arrays)
for (const item of items) {
    console.log(item);
}

// For...in (object keys)
for (const key in obj) {
    console.log(key, obj[key]);
}
```

**Python:**

```python
# Range-based (like classic for)
for i in range(10):      # 0 to 9
    print(i)

for i in range(2, 10):   # 2 to 9
    print(i)

for i in range(0, 10, 2): # 0, 2, 4, 6, 8
    print(i)

# Iterate over list (like for...of)
for item in items:
    print(item)

# Iterate with index
for index, item in enumerate(items):
    print(index, item)

# Iterate over dict
for key in obj:
    print(key, obj[key])

for key, value in obj.items():
    print(key, value)
```

### While Loop

**JavaScript:**

```javascript
while (condition) {
    // ...
}
```

**Python:**

```python
while condition:
    # ...
```

## üîß Functions

### Basic Function

**JavaScript:**

```javascript
function greet(name) {
    return "Hello, " + name;
}

// Arrow function
const greet = (name) => "Hello, " + name;
```

**Python:**

```python
def greet(name):
    return "Hello, " + name

# Lambda (limited to single expression)
greet = lambda name: "Hello, " + name
```

### Default Parameters

**JavaScript:**

```javascript
function greet(name = "World") {
    return "Hello, " + name;
}
```

**Python:**

```python
def greet(name="World"):
    return "Hello, " + name
```

### Keyword Arguments

Python has a powerful feature JS doesn't have natively:

```python
def create_user(name, email, age=None, active=True):
    ...

# Call with keyword arguments (any order!)
create_user(email="john@example.com", name="John")
create_user("John", "john@example.com", active=False)
```

### \*args and \*\*kwargs

**Python:**

```python
def my_func(*args, **kwargs):
    print(args)    # Tuple of positional args
    print(kwargs)  # Dict of keyword args

my_func(1, 2, 3, name="John", age=25)
# args = (1, 2, 3)
# kwargs = {"name": "John", "age": 25}
```

**JavaScript equivalent:**

```javascript
function myFunc(...args) {
    console.log(args); // Array of all args
}
```


## ‚ö†Ô∏è Common Gotchas

### 1. Indentation Matters!

```python
# Wrong - IndentationError!
if True:
print("hello")

# Correct
if True:
    print("hello")
```

### 2. True/False/None are Capitalized

```python
# Wrong
value = true      # NameError!
value = none      # NameError!

# Correct
value = True
value = None
```

### 3. No ++ or --

```python
# Wrong
i++    # SyntaxError!

# Correct
i += 1
```

### 4. Integer Division

```python
5 / 2      # 2.5 (float division)
5 // 2     # 2 (integer division)
```

### 5. String Immutability

```python
s = "hello"
s[0] = "H"    # TypeError! Strings are immutable

# Correct
s = "H" + s[1:]    # "Hello"
```

### 6. List as Default Argument (Dangerous!)

```python
# DANGEROUS - Don't do this!
def add_item(item, items=[]):
    items.append(item)
    return items

add_item(1)  # [1]
add_item(2)  # [1, 2] - What?! Same list is reused!

# CORRECT
def add_item(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items
```
