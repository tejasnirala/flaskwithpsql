### Dockerfile

A **Dockerfile** is a text file with instructions to build an image:

```dockerfile
# Start from a base image
FROM python:3.11-slim

# Set working directory
WORKDIR /app

# Copy and install dependencies
COPY requirements.txt .
RUN pip install -r requirements.txt

# Copy application code
COPY . .

# Define how to start the app
CMD ["flask", "run", "--host=0.0.0.0"]
```

### Docker Compose

**Docker Compose** orchestrates multiple containers:

```yaml
# docker-compose.yml
services:
    web:
        build: .
        ports:
            - "5500:5500"

    db:
        image: postgres:15
        environment:
            - POSTGRES_PASSWORD=secret

    redis:
        image: redis:7
```

**One command** starts everything:

```bash
docker compose up
```

## Why Use Docker?

### Benefits

| Benefit                      | Description                               |
| ---------------------------- | ----------------------------------------- |
| **Consistency**              | Same environment everywhere               |
| **Isolation**                | App doesn't affect host system            |
| **Reproducibility**          | Dockerfile = exact recipe                 |
| **Easy onboarding**          | New developer: `docker compose up` â†’ done |
| **No "works on my machine"** | Environment is in the container           |
| **Easy scaling**             | Run more container instances              |

### Without Docker

```bash
# New developer setup (painful)
1. Install Python 3.11 (hope it's available)
2. Install PostgreSQL (configure it)
3. Install Redis (configure it)
4. Create virtual environment
5. Install dependencies (hope they work)
6. Copy .env file
7. Set up database
8. Run migrations
9. Finally run the app
10. Debug environment issues for 2 hours
```

### With Docker

```bash
# New developer setup (easy)
1. Install Docker
2. git clone <repo>
3. docker compose up
4. App is running âœ…
```

## Essential Commands

### Container Lifecycle

```bash
# Build image from Dockerfile
docker build -t myapp .

# Run container from image
docker run myapp

# Run with port mapping
docker run -p 5500:5500 myapp

# Run in background (detached)
docker run -d myapp

# Stop container
docker stop <container_id>

# Remove container
docker rm <container_id>
```

### Inspection

```bash
# List running containers
docker ps

# List all containers (including stopped)
docker ps -a

# List images
docker images

# View container logs
docker logs <container_id>

# Follow logs (like tail -f)
docker logs -f <container_id>

# Execute command in running container
docker exec -it <container_id> bash
```

### Docker Compose

```bash
# Start all services
docker compose up

# Start in background
docker compose up -d

# Stop all services
docker compose down

# View logs
docker compose logs -f

# Rebuild images
docker compose build

# Run one-off command
docker compose run --rm web pytest
```

### Cleanup

```bash
# Remove stopped containers
docker container prune

# Remove unused images
docker image prune

# Remove everything unused
docker system prune

# Nuclear option: remove everything
docker system prune -a --volumes
```

## Docker with Flask

### Common Patterns

#### Development vs Production

```yaml
# docker-compose.yml (Development)
services:
    web:
        build:
            target: development # Use dev stage
        volumes:
            - .:/app # Mount code for hot-reload
        environment:
            - FLASK_DEBUG=1
        command: flask run --reload
```

```yaml
# docker-compose.prod.yml (Production)
services:
    web:
        build:
            target: production # Use prod stage
        # No volume mount (code baked in)
        environment:
            - FLASK_DEBUG=0
        command: gunicorn ...
```

#### Database Dependencies

```yaml
services:
    web:
        depends_on:
            db:
                condition: service_healthy # Wait for DB

    db:
        healthcheck:
            test: ["CMD", "pg_isready"]
            interval: 5s
```

#### Environment Variables

```yaml
services:
    web:
        environment:
            - DATABASE_URL=${DATABASE_URL} # From .env
            - SECRET_KEY=${SECRET_KEY}
        env_file:
            - .env # Load all from file
```

## Summary

### Key Takeaways

1. **Images are blueprints**, containers are running instances
2. **Dockerfile** defines how to build an image
3. **Docker Compose** orchestrates multiple containers
4. **Volumes** persist data and enable hot-reload
5. **Containers are lightweight** compared to VMs

### Quick Reference

| I want to...    | Command                              |
| --------------- | ------------------------------------ |
| Start the app   | `docker compose up`                  |
| Stop the app    | `docker compose down`                |
| View logs       | `docker compose logs -f`             |
| Run tests       | `docker compose run --rm web pytest` |
| Enter container | `docker compose exec web bash`       |
| Rebuild image   | `docker compose build`               |
